================================================================================
                        PROJECT OVERVIEW
================================================================================

EDU TRACK PRO is a production-ready IoT-based smart attendance management 
system that uses RFID (Radio Frequency Identification) technology to automate
the process of tracking student attendance. The system eliminates manual
attendance marking, reduces errors, and provides real-time data analytics.

================================================================================
                        SYSTEM ARCHITECTURE
================================================================================

The system consists of THREE main components:

1. HARDWARE LAYER (Arduino/NodeMCU)
   ├── ESP8266 NodeMCU Wi-Fi Development Board
   ├── MFRC522 RFID Reader Module
   ├── 16x2 LCD Display (I2C Interface)
   ├── Buzzer (Audio Feedback)
   ├── LED Indicators (Visual Feedback)
   ├── Push Button (Wi-Fi Configuration Reset)
   └── RFID Cards/Tags (Assigned to Students)

2. CLOUD LAYER (Google Sheets + Apps Script)
   ├── Google Spreadsheet Database
   │   ├── Database Sheet (Student Info: UID, Name, Roll No, Class, Address)
   │   └── Attendance Sheet (Logs: Date, Time, UID, Name, Roll No, Class, Address)
   └── Google Apps Script (Web App for receiving POST requests)

3. WEB APPLICATION LAYER (This Project)
   ├── Landing Page (index.html)
   ├── Authentication System (JavaScript-based)
   ├── Admin Dashboard (Analytics & Reports)
   └── Student Portal (Personal Attendance Calendar)

================================================================================
                        WORKING MECHANISM
================================================================================

STEP 1: RFID Card Scanning
   - Student taps their RFID card on the MFRC522 reader
   - The reader extracts the unique UID (User Identification) from the card
   - LCD displays "Scanning..." and buzzer provides audio feedback

STEP 2: Data Processing (NodeMCU)
   - ESP8266 receives the UID from RFID reader via SPI protocol
   - Creates a JSON payload with UID and timestamp
   - Connects to Wi-Fi and sends HTTPS POST request to Google Apps Script

STEP 3: Cloud Storage (Google Sheets)
   - Apps Script receives the POST request
   - Looks up student information from Database sheet using UID
   - Logs attendance record with: Date, Time, UID, Name, Roll No, Class, Address
   - Returns success/failure response to NodeMCU

STEP 4: Feedback to User
   - LCD displays student name and "Attendance Marked!"
   - Green LED lights up for success, Red for failure
   - Buzzer plays success/error tone

STEP 5: Web Dashboard Access
   - Data is fetched from Google Sheets via public JSON endpoint
   - Web app displays attendance data with analytics and visualizations
   - Admin sees all students' data; Students see only their own records

================================================================================
                        AUTHENTICATION SYSTEM
================================================================================

Type: JavaScript-based client-side authentication (Demo purposes)

Admin Login:
  - Username: admin
  - Password: admin123 (configurable)

Student Login:
  - Roll Number (from Database sheet)
  - UID (RFID card number) as password

Session Management:
  - LocalStorage based session tokens
  - Auto logout after inactivity
  - Role-based access control (admin/student)

Note: For production, implement server-side authentication with proper
      encryption and secure token management.

Frontend:
